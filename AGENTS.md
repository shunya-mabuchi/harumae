# AGENTS.md

## プロジェクト概要

「AIまえチェック」は、ChatGPT / Claude / Gemini などのLLMサービスへ文章を送る前に、個人情報・秘密情報・APIキー・社外秘っぽい内容をブラウザ内で検出し、安全化を促すChrome拡張です。

単なるAIチャットではなく、LLMに送信される前のDLPレイヤーとして動作させます。サイドパネルや独自入力欄を主役にせず、対象サイトの通常入力体験をできるだけ維持します。

## 絶対に守る方針

- UI、README、ドキュメントは日本語で書く。
- コメントも基本的に日本語で書く。
- 外部LLM APIを使わない。
- OpenAI API、Claude API、Gemini APIなどへユーザー本文を送らない。
- ユーザー本文を扱うバックエンドを作らない。ルール配信・署名検証サーバーは例外だが、配信するのは検出ルールだけで、ユーザー本文を受け取らない。
- ユーザーの貼り付け本文、送信本文、ファイル本文を永続保存しない。
- 検出結果、placeholderMap、送信履歴を永続保存しない。
- ユーザー本文を `console.log` で出力しない。
- エラー文やログにユーザー本文を含めない。
- Analyticsやトラッキングを入れない。
- 保存してよいのは設定だけ。保存先は `chrome.storage.local`。
- WebLLMモデルキャッシュはブラウザ管理下の保存領域として扱い、ユーザー本文とは分けて説明する。
- 初期実装で `<all_urls>` を無条件に要求しない。
- public repository前提で、Issue / PR / README / テストデータに実在の個人情報、実APIキー、実トークンを入れない。

## 対象サイト方針

- 初期対象は ChatGPT / Claude / Gemini。
- Perplexityは後続adapterで対応する。
- 対象サイトはadapterで分離し、DOM変更に追従しやすくする。
- サイドパネルや大きな独自入力UIではなく、通常入力欄、risk badge、送信前確認モーダルを中心にする。

## DLPポリシー

- ルールベース検出を最優先にする。
- APIキー、秘密鍵、JWT、`.env`、DATABASE_URL、トークン類、webhook URLなどはSecret Guard対象として扱う。
- `high` / `critical` とSecret Guard対象は、安全化なしでは送信不可。
- `medium` は詳細確認後にユーザーが許可可能。
- LLM結果は確定扱いせず、候補として扱う。
- 「完全に安全」「ゼロリスク」「100%検出」といった表現を避ける。

## 実装責務

- 検出エンジンは `packages/core` に集約する。
- マスキング、risk score、policy、transform modelは `packages/core` に置く。
- WebLLM関連は `packages/llm` に集約する。
- 署名付きルール配信APIは `apps/worker` に置き、署名検証とリモートルールのDetectorRule化は `packages/core` に置く。
- WebLLMは文脈チェック専用に使う。依頼文生成や最終的な安全判定には使わない。
- メール、電話番号、APIキーなどの確定情報検出をWebLLMの主役にしない。
- 拡張機能とデモサイトは同じcore検出エンジンを使う。
- WebLLMが失敗してもルールベース検出は利用できるようにする。
- 変換後テキストは再スキャンし、Secret Guard対象が残っていないか確認する。

## 型と品質

- TypeScriptの型安全性を重視する。
- `any`は禁止。どうしても必要な場合は理由をコメントに残す。
- UIから独立したロジックにはテストを書く。
- 実装後に `pnpm test`、`pnpm typecheck`、必要に応じて `pnpm build` を実行する。
- demoの主要導線は可能な範囲でPlaywrightで確認する。

## GitHub運用

- 作業は必ずIssue起点で整理する。
- Issue番号に紐づくブランチを作る。
- 変更はPRにまとめる。
- PR本文に変更内容、確認コマンド、未完了点を書く。
- 確認後にマージする。
- リポジトリはpublic前提として扱い、公開されて困る情報を残さない。

## 使ってよい表現

- 貼り付け本文は外部サーバーに送信されません。
- 検出とAI文脈チェックはユーザーのブラウザ内で実行されます。
- WebLLMの初回利用時には、ローカル推論用のモデルファイルを取得する場合があります。
- モデル取得後はブラウザキャッシュを利用します。
- あなた自身の推論サーバーや外部LLM APIは利用しません。
- 情報漏洩を完全に防ぐものではなく、消し忘れに気づくための補助ツールです。

## 避ける表現

- 完全に通信しません。
- 絶対安全です。
- 100%検出できます。
- すべての情報漏洩を防ぎます。
- ゼロリスクです。
- 生データが対象サイトに一切見えません。

## 実装後チェック

- `pnpm test`
- `pnpm typecheck`
- `pnpm build`
- 必要に応じて `pnpm test:e2e`
- READMEとUIの表現が誇大になっていないか確認する。
- ユーザー本文を保存・ログ出力していないか確認する。
- Secret Guard対象が安全化なしで送信可能になっていないか確認する。

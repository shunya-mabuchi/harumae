# AGENTS.md

## プロジェクト概要

「貼るまえ」は、ChatGPT・Claude・Gemini・Perplexity などに文章を貼り付ける前に、個人情報・秘密情報・APIキー・社外秘っぽい内容の消し忘れに気づくための Chrome 拡張です。

単なるAIチャットではなく、AIに送る前の最後の確認レイヤーとして動作します。

## 絶対に守る方針

- UIとREADMEは日本語で書く。
- コメントも基本的に日本語で書く。
- 外部LLM APIを使わない。
- OpenAI API、Claude API、Gemini APIなどへ本文を送らない。
- バックエンドを作らない。
- ユーザーの貼り付け本文を永続保存しない。
- placeholderMapを永続保存しない。
- ユーザー本文を`console.log`で出力しない。
- エラー文やログにユーザー本文を含めない。
- Analyticsやトラッキングを入れない。
- 保存するのは設定だけ。保存先は`chrome.storage.local`。
- 権限は必要最小限にする。初期実装で`<all_urls>`を要求しない。

## GitHub運用

- コードは必ずGitHubで管理する前提で進める。
- リポジトリは原則publicで作成・運用する。公開される前提で、秘密情報、個人情報、実トークン、実APIキーを絶対に含めない。
- 新しい作業はIssue起点で整理する。
- 実装はブランチを切ってPRにまとめる。
- レビュー可能な単位でPRを作成し、確認後にマージする流れを基本にする。
- GitHub remote、認証、権限が未設定の場合は、ローカルgit履歴を整えたうえで、できなかった理由と次に必要な操作を報告する。

## 実装の責務分離

- 検出エンジンは`packages/core`に集約する。
- マスキング処理と型定義も`packages/core`に置く。
- WebLLM関連は`packages/llm`に集約する。
- 拡張機能とデモサイトは同じcore検出エンジンを使う。
- WebLLMは文脈チェック専用。メール、電話番号、APIキーなどの確定情報検出の主役にしない。
- ルールベース検出を優先する。
- LLM結果は候補扱いにし、確定や安全保証として扱わない。

## 型と品質

- TypeScriptの型安全性を重視する。
- `any`は禁止。どうしても必要な場合は理由をコメントに残す。
- UIから独立したロジックにはテストを書く。
- 実装後に`pnpm test`、`pnpm typecheck`、`pnpm build`を実行する。
- demoの主要導線は可能な範囲でPlaywrightで確認する。

## 表現上の注意

使ってよい表現:

- 貼り付け本文は外部サーバーに送信されません
- 検出とAI文脈チェックはユーザーのブラウザ内で実行されます
- WebLLMの初回利用時には、ローカル推論用のモデルファイルを取得する場合があります
- モデル取得後はブラウザキャッシュを利用します
- あなた自身の推論サーバーや外部LLM APIは利用しません

避ける表現:

- 完全に通信しません
- 絶対安全です
- 100%検出できます
- すべての情報漏洩を防ぎます

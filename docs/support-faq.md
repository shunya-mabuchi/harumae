# サポートFAQと既知の制限

AIまえチェックの問い合わせ、自己診断、既知の制限をまとめます。公開Issueや問い合わせでは、貼り付け本文、送信本文、実APIキー、実トークン、実個人情報、顧客名、案件名を送らないでください。再現には実在しないダミー情報を使ってください。

## 問い合わせ前に確認すること

- 対象サイトが ChatGPT / Claude / Gemini / Perplexity のいずれかである
- AIまえチェックがOptions Pageで有効になっている
- 対象サイトごとの設定が有効になっている
- 検出ルールが無効化されていない
- 拡張を更新した後、対象サイトのタブを再読み込みしている
- WebLLMを使う場合、通常ウィンドウで確認している
- WebLLMを使う場合、Chromeの `chrome://gpu` でWebGPUが利用可能か確認している

## 問い合わせ時に書いてほしい情報

- 利用している対象サイト
- 発生した操作: 貼り付け、送信ボタン、Enter送信、ファイル添付など
- 表示された日本語メッセージ
- Chromeのバージョン
- OS
- AI文脈チェックの場合は、WebGPUの利用可否
- 拡張機能のバージョン

Options Pageの「サポート用診断情報」から、本文を含まない診断情報を作成・コピーできます。内容を確認してからGitHub Issuesへ貼り付けてください。

本文や実キーは送らず、必要なら次のようなダミーに置き換えてください。

```text
taro@example.com
090-1234-5678
GITHUB_TOKEN=ghp_dummyDummyDummyDummyDummyDummy123456
```

## FAQ

### 本文は保存されますか？

保存しません。貼り付け本文、送信本文、添付前に読み取ったテキスト本文、検出結果、placeholderMap、送信履歴は永続保存しません。保存するのは拡張機能の設定と、最後に検証済みの署名付きリモートルールキャッシュだけです。リモートルールキャッシュには本文や検出結果を含めません。

### 本文は開発者のサーバーへ送信されますか？

送信しません。ルールベース検出はブラウザ内で実行します。WebLLMによるAI文脈チェックも、ユーザーのブラウザ内で実行します。

### WebLLMが動かないのはなぜですか？

主な理由は、WebGPU非対応、GPUドライバやDawn backendの制限、シークレットモードの保存容量制限、モデル配信元への接続ブロック、初回モデル取得の失敗です。WebLLMが失敗しても、ルールベース検出は引き続き利用できます。

### 「No available WebGPU adapters」と表示されます。

ChromeがWebLLMを動かせるWebGPUアダプタを取得できていない状態です。Chromeの完全再起動、通常ウィンドウでの再確認、`chrome://gpu` のDawn Info確認、GPUドライバ更新を試してください。この状態はモデルIDの変更だけでは解消しない場合があります。

### 初回ロードが長いのはなぜですか？

WebLLMは外部LLM APIを使わずブラウザ内で推論するため、初回利用時にローカル推論用モデルファイルを取得する場合があります。モデル取得後はブラウザキャッシュやブラウザ管理下の保存領域を利用しますが、ブラウザや端末環境により再取得や再初期化が必要になることがあります。

### 対応サイトはどこですか？

対象サイトは ChatGPT / Claude / Gemini / Perplexity です。`<all_urls>` は無条件に要求しません。

### AI文脈チェックだけで安全判定できますか？

できません。AI文脈チェックは、顧客名候補、人名候補、案件名候補、契約・採用・給与・法務などの文脈上注意したい候補を補助的に表示する機能です。最終的な判断はユーザーが行います。

### 検出された内容をそのまま送れますか？

中リスクは詳細確認後にユーザー判断で送信できる設計です。高リスク、critical、秘密情報保護対象は、安全化なしでは送信できない方針です。

### ファイル添付前チェックは何に対応していますか？

MVPではテキスト系ファイルのみ対応します。PDF、docx、xlsx、画像OCRは解析しません。詳しい検討方針は [file-inspection-roadmap.md](file-inspection-roadmap.md) にあります。

### 不具合報告に本文を貼ってもよいですか？

貼らないでください。公開IssueやPRには、実在の本文、実APIキー、実トークン、実個人情報、顧客名、案件名を書かないでください。再現にはダミー情報を使ってください。

### サポート用診断情報には何が含まれますか？

拡張バージョン、Chrome/OS概要、WebGPU可否、対象サイト設定、WebLLMモデルID、ルール配信keyIdなどです。貼り付け本文、送信本文、検出文字列、placeholderMap、現在のページURLは含めません。

## 既知の制限

- 情報漏洩を完全に防ぐものではありません
- 検出漏れや誤検出が発生する可能性があります
- WebLLMの結果は候補提示であり、確定判定ではありません
- WebGPU非対応環境ではAI文脈チェックを利用できない場合があります
- WebLLMの初回利用時にはモデルファイル取得に時間がかかる場合があります
- 対象サイトのDOM変更によりadapterが動かなくなる可能性があります
- 対象サイトは ChatGPT / Claude / Gemini / Perplexity です
- ファイル添付前チェックはテキスト系ファイルのみが対象です
- PDF / docx / xlsx / 画像OCRはMVPでは解析しません
- 対象サイト独自のドラッグ&ドロップ添付やクリップボード経由のファイル添付は動作保証の対象外です

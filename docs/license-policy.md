# ライセンスと素材利用方針

AIまえチェックはpublicリポジトリとして公開しています。コード、ドキュメント、画像素材で扱いを分けます。

## コード

このリポジトリ内のソースコード、設定ファイル、テスト、ビルドスクリプトは、特に別記がない限りMIT Licenseで提供します。

対象例:

- `apps/`
- `packages/`
- `functions/`
- `scripts/`
- `.github/workflows/`
- 設定ファイル

## ドキュメント

READMEと `docs/` 配下の文章は、特に別記がない限りMIT Licenseで提供します。引用や再利用時は、リポジトリ名または作者名を示すことを推奨します。

## ブランド素材と画像

次の素材は、コードと同じMIT Licenseの対象として扱いません。

- ロゴ、アイコン、ストア掲載用プロモーション画像
- Chrome Web Store用スクリーンショット
- READMEに掲載している実画面スクリーンショット
- `docs/assets/store/` 配下の画像

これらは、AIまえチェックの紹介、レビュー、採用選考、ポートフォリオ確認、記事内での引用など、製品を説明する目的での表示は可能です。ただし、別プロダクトのロゴ、ストア素材、広告素材としてそのまま再利用することは避けてください。

デモやテストに含まれるメールアドレス、電話番号、APIキー風文字列、会社名、人物名、プロジェクト名はダミーです。実在の秘密情報や個人情報を意図していません。

## 依存関係とWebLLMモデル

依存ライブラリ、WebLLM、WebLLMが取得するモデルファイルは、それぞれの配布元ライセンスに従います。このリポジトリのMIT Licenseは、第三者ライブラリや第三者モデルのライセンスを上書きしません。

モデルや依存関係を追加・変更する場合は、ライセンス、商用利用可否、配布元、WebLLM prebuilt対応、ブラウザ内実行の安定性を確認します。WebLLMモデルの選定理由と確認手順は [webllm-model-policy.md](webllm-model-policy.md) にまとめています。

依存関係アップデート時の確認コマンド、`pnpm licenses list --json` の使い方、PRチェックリストは [dependency-maintenance.md](dependency-maintenance.md) にまとめています。

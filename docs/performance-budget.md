# ローカルDLPランタイム性能基準

AIまえチェックは、貼り付け前・送信前のユーザー操作を一度止めるため、検出精度だけでなく待ち時間の短さを重要な品質として扱います。

この文書の数値は「絶対保証」ではなく、0.1.x時点の設計目標とレビュー基準です。端末性能、Chromeの状態、対象サイトのDOM変更、WebGPU、モデル取得状況によって実時間は変わります。

## 基本方針

- ルールベース検出を主判定とし、貼り付け・送信操作の直後に同期的に実行する
- モーダル表示はWebLLMのモデルロードや推論完了を待たない
- WebLLMは追加の文脈候補を出す補助機能として扱う
- WebLLMが遅い、失敗する、利用できない場合でもルールベース検出と安全化は継続する
- 性能測定では、実ユーザー本文、検出文字列、placeholderMap、現在のページURLを保存しない
- 本文や検出結果を `console.log` しない

## 目標値

| 対象 | 目標 | 補足 |
| --- | ---: | --- |
| ルールベース検出 | 通常入力で100ms以内 | 2,000文字程度の貼り付け・送信本文を想定する |
| ルールベース検出 | 長めの入力で250ms以内 | 10,000文字程度のダミー本文を想定する |
| モーダル表示 | WebLLMを待たずに表示 | ルール検出結果だけで先に判断できる状態にする |
| WebLLM bridge起動待ち | 15秒以内で失敗扱い | `BRIDGE_LOAD_TIMEOUT_MS` の現在値 |
| WebLLM入力文字数 | 最大6,000文字 | `DEFAULT_MAX_INPUT_CHARS` の現在値 |
| WebLLM候補数 | 最大12件 | `DEFAULT_MAX_CANDIDATES` の現在値 |
| LLM候補surface | 最大80文字 | 長すぎる候補は誤読や過剰マスクを避けるため捨てる |

## 待ち方のルール

### ルールベース検出

貼り付け・送信イベントでは、まず `packages/core` の検出を実行します。高リスク、秘密情報保護対象、中リスクの判定はこの結果をもとに即時に行います。

ルール検出で検出がある場合は、WebLLMの準備状況に関係なく確認モーダルを表示します。WebLLMが未準備でも、ユーザーは安全化、キャンセル、許可可能な中リスクの確認を進められます。

### WebLLM文脈チェック

WebLLMは「追加候補を見る」ための補助です。初回はモデルファイル取得とWebGPU初期化が発生するため、即時完了を前提にしません。

0.1.xでは手動実行を基本とし、今後 `WebLLM準備済み時だけ自動実行する` 方針へ進めます。自動実行を入れる場合も、モーダル初期表示やルールベースの安全化判断はWebLLMを待ちません。

### ContextBuilder導入後の基準

将来の `ContextBuilder` では、全文をそのままWebLLMへ渡すのではなく、ルール検出箇所と文脈ヒント周辺のwindowだけを渡します。

- WebLLMへ渡す合計文字数は6,000文字以内を上限にする
- 優先順位は、高リスク周辺、秘密情報保護対象周辺、中リスク周辺、文脈ヒント周辺の順にする
- 同じ範囲が重なる場合は結合し、重複した本文を増やさない
- 入力短縮後も、ユーザー本文は保存しない

## 遅い環境での表示

WebLLMの準備や実行が遅い場合は、次のように扱います。

- ロード中: `ローカルAIモデルを準備しています。初回のみ時間がかかる場合があります。`
- 実行中: `文脈リスクを確認しています。`
- 失敗時: `ルールベースの検出結果は引き続き利用できます。`
- WebGPU非対応時: 端末やブラウザではAI文脈チェックを利用できないことを説明する

失敗時の診断情報にも、本文、検出文字列、placeholderMap、現在のページURLを含めません。

## 確認コマンド

性能方針ドキュメントの必須項目は次で確認します。

```bash
pnpm qa:performance-budget
```

ルールベース検出のローカル目安は、ダミー本文だけを使う次のコマンドで確認できます。

```bash
pnpm bench:rules
```

`bench:rules` は端末差が大きいためCIの合否条件にはしません。PRで大きな検出ロジック変更を行う場合に、変更前後の目安として使います。

## 記録してよい情報

- 実行日
- OSとChrome概要
- 拡張バージョン
- ルール検出の平均時間、p95時間
- 入力文字数
- 使ったのがダミー本文であること

## 記録しない情報

- 実ユーザー本文
- 実在する個人情報
- 実APIキーや実トークン
- 検出文字列そのもの
- placeholderMap
- 現在のページURL

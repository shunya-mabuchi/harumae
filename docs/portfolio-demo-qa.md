# ポートフォリオ用デモサイトQA

このドキュメントは、AIまえチェックの紹介用ランディングページ兼ミニデモ画面を、転職ポートフォリオとして見せる前に確認するためのメモです。プロダクト本体はChrome拡張であり、デモ画面は拡張を入れる前に価値を理解するための補助体験として扱います。

## 確認対象

- URL: ローカルの `apps/demo`
- 確認日: 2026-06-17
- 確認コマンド: Playwrightで一時的にVite dev serverを起動し、`#demo` に遷移してサンプル検出後の画面を撮影
- テストデータ: 実在しないダミー情報のみ

## 自己レビュー

### 1440px

- ファーストビューで、プロダクト本体がChrome拡張であること、主要コピー、CTA、確認モーダルの利用イメージが見える。
- Heroの主CTAは公開前の導入手順へ向き、ミニデモは補助体験として次点の導線になっている。
- 「公開前の試し方」で、Chrome Web Store公開準備中であること、GitHub確認、ローカル読み込み、対象サイトQAの順序が分かる。
- 検出対象カードは入力欄風ではなく、カテゴリ別カードとして読める。
- デモ部分は「拡張を入れる前のミニデモ」として、入力、検出結果、安全化結果が横3カラムで並び、操作と結果の関係が追いやすい。
- 高・中・低の数字だけでなく、リスクメーター、カテゴリチップ、検出項目チェックボックスがあり、マスク対象を選べる。
- ミニデモ上部の3ステップと、仕組みセクションの5ステップで、拡張機能としての操作とプロダクト構造の両方を説明している。
- フッターだけが急に黒くならず、全体の淡い背景と白いサーフェスに馴染む。

### 390px

- ヒーロー、公開前の試し方、検出対象、ミニデモ、仕組み、技術、プライバシーが1カラムで崩れずに読める。
- 公開前の導入手順カードが縦に並び、Chrome拡張が本体であることとミニデモの位置づけが混ざらない。
- ミニデモのボタン群は縦積みになり、テキストのはみ出しや重なりはない。
- ミニデモUIは入力、検出結果、安全化結果の順で縦に並び、操作順と表示順が一致している。
- 検出項目チェックボックスと安全化結果はモバイル幅でも確認できる。
- モバイルでは縦に長くなるが、情報を削りすぎず、Chrome拡張の理解につながるポートフォリオ確認用のミニデモとして成立している。

## 公開方針

初期公開はCloudflare Pagesを第一候補にする。Viteの静的ビルドを無料枠で配信しやすく、将来的にCloudflare Workersでルール配信・署名検証サーバーを追加する場合にも説明しやすいため。

GitHub Pagesは代替候補とする。GitHubだけで完結できる利点はあるが、将来のWorker連携やプレビュー運用を考えるとCloudflare Pagesの方が今回のポートフォリオ方針に合う。

Chrome Web Store公開前は、LP上で「公開準備中」と明示し、GitHubとローカル読み込み手順へ誘導する。公開後は、Heroと公開前導線セクションの主CTAをChrome Web Storeの追加リンクへ差し替える。

- WebLLM実モデルロードは端末のWebGPU対応やモデル保存容量に依存するため、ポートフォリオ公開前に実機で別途確認する。
- 公開URL確定後、Cloudflare Pages上で同じ操作を再確認する。

このドキュメントは、AIまえチェックの紹介用ランディングページ兼ミニデモ画面を、転職ポートフォリオとして見せる前に確認するためのメモです。プロダクト本体はChrome拡張であり、デモ画面は拡張を入れる前に価値を理解するための補助体験として扱います。

- テストデータ: 実在しないダミー情報のみ

## 自己レビュー

### 1440px

- ファーストビューで、プロダクト本体がChrome拡張であること、主要コピー、CTA、確認モーダルの利用イメージが見える。
- Heroの主CTAはChrome Web Store追加へ向き、ミニデモは補助体験として次点の導線になっている。
- 「拡張機能として使う」で、Chrome Web Store公開中であること、GitHub確認、対象サイトQAの順序が分かる。
- 検出対象カードは入力欄風ではなく、カテゴリ別カードとして読める。
- デモ部分は「拡張を入れる前のミニデモ」として、入力、検出結果、安全化結果が横3カラムで並び、操作と結果の関係が追いやすい。
- 高・中・低の数字だけでなく、リスクメーター、カテゴリチップ、検出項目チェックボックスがあり、安全化対象を選べる。
- ミニデモ上部の3ステップと、仕組みセクションの5ステップで、拡張機能としての操作とプロダクト構造の両方を説明している。
- フッターだけが急に黒くならず、全体の淡い背景と白いサーフェスに馴染む。

### 390px

- ヒーロー、拡張機能として使う導線、検出対象、ミニデモ、仕組み、技術、プライバシーが1カラムで崩れずに読める。
- 公開後の導入カードが縦に並び、Chrome拡張が本体であることとミニデモの位置づけが混ざらない。
- ミニデモのボタン群は縦積みになり、テキストのはみ出しや重なりはない。
- ミニデモUIは入力、検出結果、安全化結果の順で縦に並び、操作順と表示順が一致している。
- 検出項目チェックボックスと安全化結果はモバイル幅でも確認できる。
- モバイルでは縦に長くなるが、情報を削りすぎず、Chrome拡張の理解につながるポートフォリオ確認用のミニデモとして成立している。

## 公開方針

公開先はCloudflare Pagesです。Viteの静的ビルドを無料枠で配信しやすく、署名付きルール配信APIも同じCloudflare Pages Functions上で説明できるためです。

GitHub Pagesは代替候補です。GitHubだけで完結できる利点はありますが、Pages Functions、Preview deploy、Production rollbackを含めた運用説明まで考えると、現在の主運用はCloudflare Pagesに寄せています。

Chrome Web Store公開後は、LP上で「Chrome Web Store公開中」と明示し、Heroと導入セクションの主CTAをChrome Web Storeの追加リンクにする。GitHubとローカル読み込み手順は、実装確認や開発者向けの補助導線として残す。

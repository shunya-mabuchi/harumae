## タイトル

v0.1.1 - 署名付きルール配信と公開前QA強化

## 本文

AIまえチェック v0.1.1 は、v0.1.0の公開後に整備した署名付きルール配信、拡張E2Eハーネス、公開前QA、ストア導線の改善を含む更新版です。

Chrome Web Store:
<https://chrome.google.com/webstore/detail/idedmkfplfieijdcflcogkngplhkkecc>

紹介LP:
<https://ai-mae-check.pages.dev/>

### 主な変更

- 署名付きルール配信の本番APIを有効化
- ルール配信用の複数公開鍵ローテーションに対応
- 検証済みリモートルールの短期キャッシュとTTLを追加
- ルール配信失敗時は、本文を保存せずに同梱ルールへフォールバック
- Chrome拡張E2Eハーネスを追加し、ローカル模擬composerでpaste/submit/安全化を検証
- Chrome Web Store提出前QA、公開文書同期QA、プライバシー回帰QAを強化
- README、LP、プライバシーポリシー、サポート導線を0.1.1の説明に合わせて整理

### ルール配信

- 本番エンドポイント: `https://ai-mae-check.pages.dev/api/rules/latest`
- `keyId`: `ai-mae-check-rules-2026-06-v2`
- ルールバージョン: `2026.06.23.1`
- 拡張側は署名検証に成功したルールだけを採用します
- 署名検証失敗時、ネットワークエラー時、キャッシュ期限切れ時は同梱ルールで動作します

### プライバシー

- 貼り付け本文や送信本文は永続保存しません
- 検出結果、placeholderMap、送信履歴は永続保存しません
- `chrome.storage.local` に保存するのはユーザー設定と検証済みの署名付きリモートルールキャッシュだけです
- 外部LLM APIや開発者のサーバーへ本文を送信しません
- WebLLMの初回利用時には、ローカル推論用のモデルファイルを取得する場合があります

### 既知の制限

- 情報漏洩を完全に防ぐものではありません
- 検出漏れや誤検出が発生する可能性があります
- WebLLMによるAI文脈チェックは補助的な候補提示です
- WebGPU非対応環境ではAI文脈チェックを利用できない場合があります
- ファイル添付本文そのものの解析は0.1.1の対象外です
- 実サイトのDOM変更により、ChatGPT / Claude / Gemini / Perplexity上のadapter挙動は継続確認が必要です

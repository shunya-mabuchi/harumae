# AIまえチェック プライバシー方針

最終更新日: 2026-06-16

AIまえチェックは、ChatGPT、Claude、Geminiなどに文章を送る前に、個人情報・秘密情報・APIキーなどの消し忘れに気づくためのChrome拡張です。

## 基本方針

- 貼り付け本文、送信本文、添付前に読み取ったテキスト本文を永続保存しません。
- 貼り付け本文、送信本文、添付前に読み取ったテキスト本文を開発者のサーバーへ送信しません。
- 外部LLM API、OpenAI API、Claude API、Gemini APIなどへユーザー本文を送信しません。
- Analyticsやトラッキングを行いません。
- ユーザー本文を `console.log` などで出力しません。
- エラー表示や診断メモにユーザー本文を含めません。

Chrome Web Storeの申告上は、拡張機能が入力欄テキストをブラウザ内で検査するため、該当するユーザーデータカテゴリを扱うものとして開示します。ただし、開発者のサーバーへユーザー本文を送信・収集せず、販売・第三者提供・広告目的利用・信用判断目的利用もしません。

## 保存する情報

AIまえチェックが保存するのは、拡張機能の設定のみです。設定はChromeの `chrome.storage.local` に保存されます。

保存する設定の例:

- 拡張機能の有効/無効
- 対象サイトごとの有効/無効
- 検出ルールごとの有効/無効
- WebLLMによるAI文脈チェックの有効/無効
- WebLLMモデルID
- AI文脈チェックの実行モード

保存済み設定は、拡張機能のOptions Pageにある「設定を初期化」から削除できます。初期化すると、対象サイト、検出ルール、AI文脈チェックに関する保存済み設定が削除され、初期値に戻ります。

Chromeから拡張機能を削除した場合、Chromeが管理する拡張機能の保存領域もChrome側の仕様に従って削除されます。AIまえチェックは、貼り付け本文、送信本文、検出結果、placeholderMap、送信履歴を保存していないため、これらの削除操作で消すべき本文データは保持していません。

## 保存しない情報

次の情報は永続保存しません。

- 貼り付け本文
- 送信本文
- 添付前チェックで読み取ったファイル本文
- 検出結果
- マスキング用のplaceholderMap
- 送信履歴
- AI文脈チェックの入力本文
- AI文脈チェックの出力本文

## WebLLMについて

AI文脈チェックでは `@mlc-ai/web-llm` を利用し、検出とAI文脈チェックはユーザーのブラウザ内で実行されます。

WebLLMの初回利用時には、ローカル推論用のモデルファイルを取得する場合があります。モデル取得後はブラウザキャッシュやブラウザ管理下の保存領域を利用します。

貼り付け本文は外部サーバーに送信されません。あなた自身の推論サーバーや外部LLM APIは利用しません。ただし、モデルファイルの取得では、WebLLMのモデル配信元やブラウザのキャッシュ機構に依存します。

## ファイル添付前チェックについて

テキスト系ファイル添付前チェックでは、対応するテキストファイルをユーザーのブラウザ内で読み取り、ルールベース検出を実行します。読み取ったファイル本文は永続保存せず、開発者のサーバーや外部LLM APIへ送信しません。

MVPでは `.txt`, `.md`, `.csv`, `.json`, `.yaml`, `.yml`, `.env`, `.log`, `.js`, `.ts`, `.py`, `.go`, `.rb`, `.java`, `.html`, `.xml` などのテキスト系ファイルを対象にします。PDF / docx / xlsx / 画像OCRは解析しません。また、対象サイト独自のドラッグ&ドロップ添付やクリップボード経由のファイル添付は、サイト実装に依存するため動作保証の対象外です。

## 権限について

AIまえチェックは、初期対象サイトとして次のサイトでのみ動作します。

- `https://chatgpt.com/*`
- `https://chat.openai.com/*`
- `https://claude.ai/*`
- `https://gemini.google.com/*`

初期実装では `<all_urls>` を要求しません。

利用するChrome権限:

- `storage`: 拡張機能の設定をブラウザ内に保存するために使います。

対象サイトのhost permissions:

- 対象サイト上で貼り付け操作、送信前操作、テキスト入力欄を確認するために使います。

ルール配信を有効にしている場合でも、拡張機能がルール配信Workerへ送るのは `GET /api/rules/latest` だけです。貼り付け本文、送信本文、検出結果、placeholderMap は送信しません。外部LLM APIへ本文を送ることもありません。

## 第三者への提供

AIまえチェックは、ユーザー本文を開発者のサーバーや外部LLM APIに送信しません。したがって、ユーザー本文を第三者へ販売・共有することはありません。

## 注意

AIまえチェックは情報漏洩を完全に防ぐものではありません。検出漏れや誤検出が発生する可能性があります。最終的に送信するかどうかはユーザーが判断してください。

WebGPU非対応環境やモデル取得に失敗する環境では、AI文脈チェックを利用できない場合があります。その場合も、ルールベースの検出は引き続き利用できます。

サポートや脆弱性報告の際も、貼り付け本文、実APIキー、実トークン、実個人情報、顧客名、案件名を公開IssueやPRへ書き込まないでください。再現には実在しないダミー情報を使ってください。

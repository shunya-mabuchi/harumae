# WebLLM 実機確認メモ

WebLLMは、正規表現だけでは拾いにくい文脈上の注意候補をブラウザ内で確認するために使います。外部LLM APIや開発者の推論サーバーは使いません。

端末別、OS別、Chrome状態別の確認結果は [WebLLM対応環境とモデル互換性マトリクス](webllm-compatibility-matrix.md) の形式で記録します。IssueやPRへは、ユーザー本文、実APIキー、実トークン、実在の個人情報、placeholderMapを貼らないでください。

## 成功時

確認観点:

- AI文脈チェック開始時に「ローカルAIモデルを準備しています。初回のみ時間がかかる場合があります。」が表示される
- 初回はモデルファイル取得に時間がかかる場合がある
- モデル取得後はブラウザキャッシュやブラウザ管理下の保存領域を利用する
- 候補がある場合は、顧客名候補、人名候補、案件名候補などが「候補」として表示される
- 候補なしの場合は「安全を保証するものではありません」と表示される
- WebLLM候補は自動で確定扱いせず、ユーザーが確認できる

## WebGPU非対応時

表示される想定メッセージ:

> このブラウザまたは端末ではAI文脈チェックを利用できません。ルールベースの検出は引き続き利用できます。

確認観点:

- ルールベース検出は利用できる
- マスク・安全化の操作は継続できる
- エラーメッセージにユーザー本文を含めない
- モデル変更で解消しない場合があることを案内する

## モデル取得失敗時

表示される想定メッセージ:

> ローカルAIモデルの取得に失敗しました。モデル配信元への接続がブロックされている可能性があります。ルールベースの検出結果は引き続き利用できます。

確認観点:

- Hugging FaceやGitHub rawなどのモデル配信元への接続制限を確認する
- 広告ブロッカー、セキュリティソフト、社内ネットワーク制限の影響を疑う
- ルールベース検出結果が消えない
- エラーにユーザー本文を含めない

## 保存領域不足時

private browser / シークレットモードでは、モデル保存領域が不足し `QuotaExceededError` が出る場合があります。

確認観点:

- 通常ウィンドウで再試行する
- Chrome DevTools > Application > Storageから該当サイトデータを削除して再試行する
- ディスク空き容量を確認する

## 手動確認用サンプル

```text
佐藤様向けに、Project Blue Bridge の提案メモを作成します。

まだ正式発表前なので、社外共有はしない前提でお願いします。

候補者の山田花子さんについて、最終面談後の評価メモも含めます。
給与条件は現職より少し上げる方向で、内定前に社内だけで確認したいです。
```

このサンプルは実在しないダミー情報です。
